Comparision between Get and Post Method
Get Method Post Method
Get is used to request data from a specific resources. Post is used to send data to a server to create/update a resource.
Get request remains in the browser history. Post request do not remains in the browser history.
Get request have length restrictions. Get request have no restrictions on data length.
In Get data is visible to everyone in the url. In Post Data is not displayed in url.
Get is less secure campare to Post because data sent is part of the url. post is safe because the parameter are not store in the browser history.
It is very easy to bookmark data using Get mothod. Post requested cannot be bookmarked.
Get requested can be cacheable. The Post request is hardly cacheable.
This method supports only string data types. This method supports different data types, such as string,numeric,binary,etc..
It enables you to save the results of a HTML form. You can send user-generated data to the web server.
You can easily use GET method to request required data This method can be used to send binary as well as ASCII data
Easier to hack for script kiddies. More difficult to hack.
GET method should not be used when sending passwords or other sensitive information. POST method used when sending passwords or other sensitive information.
